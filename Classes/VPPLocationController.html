Overview

VPPLocation Library simplifies the task of retrieving the user location and geocoder info about it.

In order to use it you should implement:

  • VPPLocationControllerLocationDelegate to receive updates of the user location.
  • VPPLocationControllerGeocoderDelegate to receive updates of information about user location.

Once implemented just add your class as delegate, using the methods addLocationDelegate: and addGeocoderDelegate:.

Important This library depends on CoreLocation framework, MapKit framework and SynthesizeSingleton library (by Matt Gallagher).

Important MKReverseGeocoder has been deprecated with the new CLGeocoder class included in iOS 5. Check out http://developer.apple.com/library/ios/#documentation/MapKit/Reference/MKReverseGeocoder_Class/Reference/Reference.html and http://developer.apple.com/library/ios/#documentation/CoreLocation/Reference/CLGeocoder_class/Reference/Reference.html#//apple_ref/occ/cl/CLGeocoder for further information. Take into account that this library still uses MKReverseGeocoder to give support to iOS 4.

desiredLocationAccuracy

Indicates the desired location accuracy.

@property (nonatomic, assign) CLLocationAccuracy desiredLocationAccuracy

Discussion

CLLocationAccuracy values are:

  • kCLLocationAccuracyBestForNavigation
  • kCLLocationAccuracyBest
  • kCLLocationAccuracyNearestTenMeters
  • kCLLocationAccuracyHundredMeters
  • kCLLocationAccuracyKilometer
  • kCLLocationAccuracyThreeKilometers

Important Note that this accuracy is not guaranteed, and that the better the accurary, the faster the battery gets drained.

strictMode

Indicates if location updates must be strict.

@property (nonatomic, assign) BOOL strictMode

Discussion

In a nutshell, strict mode checks if points are:

  • invalid accuracy
  • out of order
  • before the manager was initialized

Check out http://troybrant.net/blog/2010/02/detecting-bad-corelocation-data/ for further information.
